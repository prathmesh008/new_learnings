## 2026-05-15 — HTTP caching headers

Okay, so HTTP caching headers. Man, they're trickier than they look. Always thought `no-cache` meant like, never ever save this response. But nope, turns out it actually means "revalidate with the server before serving it". It's not "don't cache", it's "don't serve without asking". The actual "don't even store it" one is `no-store`. Big difference. Like, `Cache-Control: public, max-age=3600, no-cache` is totally valid, but `no-store` means wipe it completely. Wild.
