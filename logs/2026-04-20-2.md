## 2026-04-20 — PostgreSQL indexing

Okay so, `EXPLAIN ANALYZE` is actually kinda mind-blowing for seeing if your indexes are even doing anything. I'd just been slapping indexes on foreign keys w/o thinking much, but turns out you gotta *check* if PG is bothering to use 'em. Ran `EXPLAIN ANALYZE SELECT * FROM orders WHERE user_id = 123;` on that new feature and it was still doing a Seq Scan. Duh, the table was tiny, so the planner was like "nah, faster to just read everything". Made me realize indexing isn't free; it adds overhead to writes. Need to be smarter about this.
