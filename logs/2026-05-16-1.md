## 2026-05-16 — React Context vs Redux

Okay so been messing with React Context vs Redux again today. I keep seeing people say "Context for simple stuff, Redux for complex" but that's so vague. What really clicked for me today is that yeah, you can totally use Context with `useReducer` and get a `dispatch` and `state` just like Redux. But it's still just passing that stuff down the tree, no real global store, no middleware like thunks, no DevTools integration outta the box. So it *looks* similar with `const [state, dispatch] = useReducer(...)` but the *infra* is way different. That's kinda wild how much you can build with just hooks.
