## 2026-05-30 — environment variables best practices

Been reading about env vars today and like, it finally clicked that they're *always* strings. Seriously, even if I set `PORT=3000`, my app gets "3000", not the number 3000. So gotta remember to `parseInt(process.env.PORT, 10)` or whatever when I need a number. That explains some weird config bugs I've had sometimes where stuff just wasn't working right. Such a simple thing but I totally overlooked it.
