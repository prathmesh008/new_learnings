## 2026-05-23 — React rendering optimization

okay so i finally get why `useCallback` is a thing for real. been seeing it around and just kinda thought it was some advanced stuff, but today it clicked hard. if you pass a function down to a child component, and that child is wrapped in `memo`, the parent re-renders and *poof*, new function reference *every single time*. `memo` sees new props, so child re-renders too even if nothing visual changed! `useCallback` memoizes the function itself, stopping that prop change. like, `const handleClick = useCallback(() => { /* do stuff */ }, [deps])` actually makes sense now for preventing dumb re-renders. Wild.
