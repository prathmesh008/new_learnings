## 2026-04-30 — environment variables best practices

Okay so I finally properly grokked the whole `.env` file thing. For ages I just thought it *was* the best practice, like, deploy that file to prod. But nah, it's just a local dev convenience, right? The real deal is setting those actual env vars directly in the production environment, like Heroku config vars or AWS Secrets Manager, not shipping a local `.env` file. That actually makes so much more sense now. No more accidentally pushing my dev secrets, lol. Definitely a lightbulb moment.
