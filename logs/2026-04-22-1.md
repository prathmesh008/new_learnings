## 2026-04-22 — HTTP caching headers

Okay so finally wrapping my head around `Cache-Control: no-cache` vs `no-store`. Been confusing those for ages, thought `no-cache` meant "don't you dare store this." But nah, it actually means "cache it, but *always* revalidate with the server." So like, you'd send `If-None-Match` on the next req. `no-store` is the real "no copy at all, ever" one. Big difference, totally changes how I'm gonna set headers for sensitive API responses.
