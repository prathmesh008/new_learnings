## 2026-05-14 — React rendering optimization

ugh, spent way too long today trying to figure out why my `ProductCard` component kept re-rendering even with `React.memo()`. Turns out, it was the `onAddToCart` prop! Every render of the parent `ProductList` was creating a *new* function reference for `onAddToCart`, completely bypassing `memo`'s prop comparison. Duh. Had to wrap `onAddToCart` in `useCallback` like `const onAdd = useCallback(() => { /* do something */ }, [dependencies])`. Suddenly, `ProductCard` was chill. Small thing but felt like a big win, saves so many useless renders.
