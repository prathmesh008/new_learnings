## 2026-06-07 — Git rebase vs merge

Man, been diving into `git rebase` today and it actually clicked why ppl rave about it for local branches. My feature branch had like 10 commits, mostly small fixes and "oops pushed wrong file". Running `git rebase -i main` let me squash all that garbage into one clean commit before I push it. Now when it merges, main's history will be so much cleaner. No more "typo fix again" commits cluttering things up. Pretty slick.
