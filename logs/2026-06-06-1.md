## 2026-06-06 — load balancing algorithms

Okay, so been diving into load balancing algorithms today, and the difference between simple round-robin and least connections totally clicked. Round-robin is just so dumb, right? Just cycles through servers even if one is already swamped and another is basically idling. But least connections, *that's* where it gets smart. It actually checks which server has the fewest active requests and sends traffic there. Like, duh, but it means the LB needs to actually keep track, not just blind-cycle. Makes so much more sense for actual real-world traffic where servers aren't always evenly loaded. An NGINX proxy pass with `least_conn;` is basically telling it to do that. Simple but powerful.
