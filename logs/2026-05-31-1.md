## 2026-05-31 — React rendering optimization

Okay, so `React.memo` finally clicked today, after hours of debugging this weird re-render. I thought if a component got the same props, it just wouldn't bother, right? But nope. If I pass an object prop like `data={{id: 1, name: 'test'}}` directly from the parent, `memo` still thinks it's a new prop *every single time*. Because it's checking the *reference*, not the actual contents. That's why it kept re-rendering my `ProductCard` even though `id` and `name` were identical. Such a subtle but massive gotcha.
