## 2026-04-26 — Docker networking

Okay so, docker networking, man. It always felt a bit... opaque. But today it finally clicked with custom networks. Like, you create one with `docker network create my-app-net`, then spin up a db container and an app container on it. And then the app can just talk to the db by its container name! Like `mysql -h db_container_name`. No more localhost headaches or trying to find IPs. So simple, makes way more sense than the default bridge.
