## 2026-05-06 — Redis data structures

Okay, so Redis hashes just *clicked* for me today. Always kinda just used SET for everything, or multiple SETs for related stuff, which felt clunky. But hashes are basically like, objects? Or dictionaries. You just `HSET user:1 name "Alice" age 30` and all those fields are grouped under one key. So much cleaner than a bunch of `user:1:name`, `user:1:email` keys. Totally get the use case now, feels obvious almost.
