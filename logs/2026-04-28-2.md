## 2026-04-28 — Docker volumes

Okay so, volumes. Man, that finally clicked today. Been messing around with Docker for a while but never really *got* why you'd use a named volume over a bind mount for actual data. Like, it's just a folder, right? But nope, the persistence part and the fact Docker manages it on the host side, especially on my M1, is a game changer for databases. `docker run -v pg_data:/var/lib/postgresql/data postgres` means I don't care where it is, Docker just handles it. Pretty sweet.
