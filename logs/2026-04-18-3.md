## 2026-04-18 — nginx reverse proxy

okay so I've been messin' with nginx reverse proxy all day and man, `proxy_pass` was seriously trippin me up. The whole trailing slash thing, like `proxy_pass http://my-app/` versus `proxy_pass http://my-app;` – that's what finally clicked. If you add the slash, nginx strips the path from the original request before sending it to the backend. Without it, it appends the full path. So like, `/api/users` would become `/users` with the slash, but stay `/api/users` without it. Small detail but huge for routing.
