## 2026-05-22 — Node.js event loop

okay so the whole `setImmediate` vs `setTimeout(0)` thing finally kinda clicked for real today, especially when inside an I/O op. I always knew they were different but running this simple test: `fs.readFile('file.txt', () => { setTimeout(() => console.log('timeout'), 0); setImmediate(() => console.log('immediate')); });` showed `immediate` consistently first. It's cuz `setImmediate` runs in the "check" phase *after* "poll" (where I/O callbacks land), while `setTimeout(0)` goes back to the "timers" phase which is *before* poll in the next tick. Mind-bending stuff.
