## 2026-06-03 — CSS specificity

Okay so specificity... that stuff is a trip. Today it finally clicked why some of my styles just *would not* apply, even when the dev tools were clearly showing my class-based rule. Turns out IDs just stomp all over everything else. Like, if you have `#my-item { color: red; }` and also `.my-item { color: blue; }` on the same div, it's always gonna be red. No contest. Pretty wild how much more weight an ID gets compared to a class selector. Makes me wanna avoid IDs in CSS entirely almost.
