## 2026-05-14 — Docker volumes

Okay so, finally grokking docker volumes a bit better. For ages I was just like, whatever, bind mounts, easy. But for *actual* app data, like my Postgres db, named volumes are just wayyy cleaner, and actually make sure your data doesn't vanish if the container gets rekt. `docker volume create myapp_db_data` then just `-v myapp_db_data:/var/lib/postgresql/data` in the run command. Makes so much more sense than hunting for host paths.
