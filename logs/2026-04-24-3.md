## 2026-04-24 — debounce vs throttle

Okay so debounce vs throttle finally clicked today, like *really* clicked. For search bar stuff, you want debounce, right? Because you only want the API call once the user *stops* typing, not every single keypress. But for something like a scroll event, throttle's the guy. It just limits how often it can fire, like `onScroll={throttledHandler}` where it runs at most once every 100ms or whatever, no matter how fast you're scrolling. So simple now, why did that take so long.
