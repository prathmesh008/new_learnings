## 2026-05-08 — PostgreSQL indexing

Okay so I've been reading about Postgres indexing and like, it finally clicked that indexes aren't just a magic "make everything fast" button. They're amazing for reads, especially when querying huge tables on specific columns, but then you pay a price on writes. Every `INSERT`, `UPDATE`, `DELETE` has to update the index too, which makes sense but I didn't really *get* it before. Used `EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'test@example.com';` to see how it picked up my new index, that was super cool.
