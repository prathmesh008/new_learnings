## 2026-06-02 — Redis data structures

Man, been playing with Redis hashes today, and it finally clicked *why* they're so handy. Like, before I was just doing `SET user:1:name "Alice"`, `SET user:1:email "alice@example.com"` right? So dumb. With `HSET user:1 id 1 name "Alice" email "alice@example.com"` it's just so much cleaner, kinda like a nested object. All user data grouped under one key, makes way more sense for user profiles or config stuff. Duh, so obvious now.
