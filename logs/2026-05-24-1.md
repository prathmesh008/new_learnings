## 2026-05-24 — HTTP caching headers

ugh, been reading about http caching headers all afternoon and finally got my head around `Cache-Control: no-cache`. like, it doesn't actually mean "don't cache this at all", which is what i always assumed. it just means the browser/proxy HAS to revalidate with the server before using it, totally different from `no-store`. felt kinda dumb for not getting that earlier lol. so if you do `res.setHeader('Cache-Control', 'no-cache')` it's still *cached*, just checked every time. that's wild.
