## 2026-05-27 — PostgreSQL indexing

Okay so, been digging into PostgreSQL indexing and man, partial indexes just *clicked*. Like, I always thought you just index everything, right? But what if you only ever query for active users? Instead of indexing the whole `users` table `status` column, you can do `CREATE INDEX active_users_idx ON users (email) WHERE status = 'active';`. Mind. Blown. Cuts down index size *and* makes lookups faster for the common case, so smart. Totally changes how I'm thinking about query perf.
