## 2026-05-01 — Node.js event loop

Okay so the Node event loop is wild. Today it finally clicked how `setImmediate` and `setTimeout(0)` interact, like, when you put 'em inside an I/O callback. Before, I always thought `setTimeout(0)` would *always* run first but nope, inside the poll phase, `setImmediate` actually runs before the `setTimeout`'s timer gets processed. Blew my mind! Saw an example where `fs.readFile` runs, then `setImmediate` almost always fires before `setTimeout`. It makes sense now why they have different phases.
