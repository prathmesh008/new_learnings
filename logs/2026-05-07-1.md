## 2026-05-07 — JavaScript closures

okay so closures finally clicked today, kinda. I always thought it was just about returning a function, but it's more like that inner function *remembers* where it was born, including all the variables around it. Even after the outer function's done, it still keeps those variables alive in its scope. Like that `makeCounter` example:

```javascript
function makeCounter() {
  let count = 0;
  return function() {
    return count++;
  };
}
const counter1 = makeCounter();
console.log(counter1()); // 0
console.log(counter1()); // 1
```

Man, that `count` variable just hangs around for `counter1` specifically, wild.
