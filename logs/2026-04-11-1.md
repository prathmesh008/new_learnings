## 2026-04-11 — environment variables best practices

Okay so, finally really wrapped my head around the whole `.env` thing today. For local dev, `dotenv` makes it so easy, just stick `process.env.API_KEY` in your code and it pulls from the file. But then I was thinking, if you don't commit that `.env` file (which everyone says DON'T!), how does it work on production?

And it clicked. You set those variables directly on the server itself, like on Netlify or Vercel, in their UI settings. So your local `.env` is literally *only* for your machine. That's why something like `DB_URL` can be totally different for dev vs prod. Duh. Seems obvious now but it was a little hazy before.
