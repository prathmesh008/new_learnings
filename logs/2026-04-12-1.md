## 2026-04-12 — Node.js event loop

Okay so, been reading about the Node.js event loop again, and man, the `setImmediate` vs `setTimeout(..., 0)` thing finally clicked a little harder. My brain always just lumped them together, like "run this ASAP." But no! Turns out they run in totally different phases. Like, if you run this:

```js
setTimeout(() => console.log('timeout'), 0);
setImmediate(() => console.log('immediate'));
```

The order isn't guaranteed, it's just race-y. But inside an I/O callback, `setImmediate` *always* runs before `setTimeout(0)`. That's the part that really messed with me, but now I get why I was seeing weird ordering sometimes. It's all about those phases, wild.
