## 2026-04-23 — API versioning strategies

Okay so been reading about API versioning, totally mind-blown how many ways there are to do it. The whole `Accept` header thing with `application/vnd.myapi.v1+json` clicked today. Like, instead of `/api/v1/users`, you just hit `/api/users` and the header tells it what version to give you. Seems way cleaner for not breaking old clients, you can keep `v1` and `v2` running on the same endpoint internally. Much less clunky than URL paths, def a better vibe for big apps.
