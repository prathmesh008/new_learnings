## 2026-05-31 — debounce vs throttle

Okay so been wrestling with debounce vs throttle, and today it kinda clicked. The big thing for debounce is that it only runs *after* a period of inactivity, like if I'm typing in a search box, it waits for me to stop for, say, `500ms` before it actually calls the API. Not just "fire every X ms". Throttle, though, that's just "run every X ms, max." So resize events, scroll stuff, that's throttle. Debounce is way more "smart" about when to actually do the work.
