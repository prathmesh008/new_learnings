## 2026-04-20 — SQL query optimization

Okay so I was looking at this super slow query today, it was driving me nuts. Kept thinking like, what's even going on under the hood? Then I remembered `EXPLAIN ANALYZE` in pg and just slapped it on. Holy crap, seeing the actual execution plan with the timings for each step, that was a game changer. Like, seeing "Seq Scan" take forever vs. "Index Scan" for my `WHERE` clause made so much more sense. Definitely gonna start using that more.
