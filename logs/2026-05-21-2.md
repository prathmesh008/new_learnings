## 2026-05-21 — TypeScript generics

ugh, generics still mess with my head sometimes. today it was like, `function identity<T extends {id: string}>(arg: T): T` – it clicked that `extends` here doesn't mean inheritance in the JS sense, more like "is compatible with" or "must have these properties." Like, I can pass in any object *as long as* it has an `id: string` property. Still feels a bit backwards but makes sense for type safety.
