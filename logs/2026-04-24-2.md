## 2026-04-24 — database transactions

Okay so today was all about db transactions, finally clicked why they're such a big deal. Like, you know how a bank transfer needs to happen completely or not at all? If money leaves account A but doesn't hit account B, that's a disaster. Transactions make sure it's atomic – all or nothing. You literally do `BEGIN; UPDATE accounts SET balance = balance - 100 WHERE id = 1; UPDATE accounts SET balance = balance + 100 WHERE id = 2; COMMIT;` and if anything goes wrong before `COMMIT` you just `ROLLBACK` and it's like nothing ever happened. So powerful for data integrity, whoa.
