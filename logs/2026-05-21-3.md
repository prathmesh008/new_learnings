## 2026-05-21 — JavaScript closures

Okay so closures. FINALLY kinda clicked today, I think. It's like, a function remembers its lexical environment, even after the outer function has finished executing. That's the key part I was missing. Been playing around with a simple counter example: `function makeCounter() { let count = 0; return function() { count++; return count; }; }` and before it just seemed like magic how `count` stayed alive. But nah, that inner function just holds onto it. So cool.
