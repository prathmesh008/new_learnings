## 2026-04-20 — environment variables best practices

Okay so, env vars... finally clicked that `dotenv` package is like, purely for local dev convenience. Was confusing me why everyone always said "don't commit .env" but then how do secrets get to prod? Duh, on prod, you set them directly in the actual environment, like on Heroku config vars or whatever. So my `process.env.API_KEY` on prod gets it from the platform, not some file. Makes so much sense now. Just gotta remember to set them for real in the deploy pipeline.
