## 2026-05-12 — debounce vs throttle

okay so finally got my head around debounce vs throttle today. Like, debounce is basically "wait for a pause," right? So for a search bar, you only fire the API request *after* they stop typing for a bit. Throttle is more like, 'do this at most once every X time,' like scroll events. Makes total sense now why I was getting weird behavior before.
