## 2026-04-16 — JWT vs sessions

Okay so JWTs vs sessions has finally kinda clicked today. I was always like, how is a JWT "stateless" if you still have to validate it? But no, the server doesn't *store* the token, just kinda verifies its signature and expiration with a secret key it already has. The client sends the whole thing, like `Authorization: Bearer <my_jwt_here>`, and the server just checks *that*. With sessions, the server literally has to look up that session ID in a DB or something every single request. That's the difference, the server's memory footprint is way less for JWTs once issued. Wild.
