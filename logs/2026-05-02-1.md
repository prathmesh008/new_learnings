## 2026-05-02 — React Context vs Redux

Okay so spent ages today looking at Context vs Redux. My brain was like, wait, why even bother with Redux if Context exists now? But then it clicked: Context is legit great for simpler stuff, like passing down a theme or auth state that doesn't change *all the time*. For anything big and complex, with lots of actions & frequent updates, Redux is still the boss. Think a massive chat app or something, you'd never try to handle that with just Context providers everywhere.
