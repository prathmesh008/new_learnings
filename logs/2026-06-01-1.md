## 2026-06-01 — Git rebase vs merge

Okay, so I finally wrapped my head around `rebase` vs `merge` today, like *really* got it. Always just used merge cause it was default, right? But rebase actually makes your commit history so much cleaner, like a straight line instead of all those merge commits cluttering everything up. The whole "rewriting history" thing still feels a bit spicy, gotta be careful on shared branches for sure. But for my own feature branches before pushing, `git rebase main` feels like the move now. My log will look so pretty!
