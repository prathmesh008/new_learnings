## 2026-04-16 — HTTP caching headers

Okay so been reading about HTTP caching headers, finally clicked today how ETag actually works with If-None-Match. The server sends an ETag, right? And next time, the browser just sends it back in `If-None-Match: "some-unique-tag"`. If the resource is unchanged, server sends back a 304 Not Modified. Boom, no full re-download. Saves so much bandwidth cause it's not re-sending the whole damn file. Wild.
