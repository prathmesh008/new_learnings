## 2026-04-23 — JavaScript closures

Okay so closures. Always kinda thought I got them but today it properly clicked, like, a function remembers the environment it was created in, even if that outer function's already finished executing. Wild. Tried that classic counter example: `function makeCounter() { let count = 0; return function() { count++; return count; }; } let myCounter = makeCounter(); console.log(myCounter()); // 1`. The `count` variable just *stays* there between calls. Mind blown, honestly. No wonder they're everywhere.
