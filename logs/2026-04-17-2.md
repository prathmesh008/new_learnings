## 2026-04-17 — TypeScript generics

Ugh, generics again. Been trying to wrap my head around them for a while and today it clicked a bit that you don't *always* have to explicitly declare the type when calling a generic function. Like, if you do `function identity<T>(arg: T): T { return arg; }`, and then call `identity("hello")`, TS just *knows* T is string. Woah. Saves so much boilerplate when it can infer it, definitely less annoying than I thought.
