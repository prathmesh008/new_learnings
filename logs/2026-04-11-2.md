## 2026-04-11 — HTTP caching headers

So turns out, `Cache-Control: no-cache` totally threw me. Always figured that meant "don't cache it, dummy." But nope, that's `no-store`. `no-cache` actually means the browser *can* stash it, but it absolutely *has* to hit the server to check if it's still fresh with `If-None-Match` or whatever, before using the cached version. It's like, a forced revalidation. So like, `Cache-Control: no-cache, max-age=0` is pretty common for dynamic stuff. The difference just finally clicked.
