## 2026-04-11 — React rendering optimization

Okay so `useCallback` finally clicked today. Was pulling my hair out on this perf issue where a child component kept re-rendering for like no reason at all. Turns out even if the function *looks* the same, React legit sees it as a new prop every single time if it's defined inside the parent on render. So `useCallback` memoizes the actual function itself, right, so the child (if it's `React.memo`'d) doesn't get a new prop reference. No re-render. Mind blown. Before: `const handleClick = () => {...}`. Now: `const handleClick = useCallback(() => {...}, [])`. Such a small change, massive implications.
