## 2026-05-17 — load balancing algorithms

okay so spent a bit today digging into load balancing algorithms again. i always kinda just hand-waved IP hashing as 'magic sticky sessions' before, like something the pros just *knew*, but it finally clicked today. it's literally just taking the client's IP and hashing it to pick a server. that way the same user always goes to the same backend. makes total sense for keeping state, super elegant even if it can skew load if one client blasts requests. guess that's why you gotta balance between sticky and even distribution.
