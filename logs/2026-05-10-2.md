## 2026-05-10 — PostgreSQL indexing

okay so been reading about PostgreSQL indexing and partial indexes clicked today. Like, you can index only *part* of your table based on a `WHERE` clause in the `CREATE INDEX` statement. So if I have a `users` table and only really care about active users for certain lookups, I can do `CREATE INDEX idx_active_users_email ON users (email) WHERE active = TRUE;`. It's like, so obvious in hindsight. Smaller index, faster updates and lookups. Genius.
