## 2026-05-06 — SQL query optimization

Okay so been banging my head against this one slow query for ages, kept thinking my indexes were broken. Turns out, it's how I was doing the `LIKE` clause! Had `WHERE description LIKE '%searchterm%'` and apparently that totally kills idx usage. Duh, how could it even use an index if it has to scan from the start of every string? Switched it to `LIKE 'searchterm%'` for a test and holy crap, instant speed. So it really matters where that wildcard goes, man, big perf hit.
